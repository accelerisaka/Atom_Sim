# 绝对严格的架构规则（必须遵守）
在编写代码时，你必须严格遵守以下规则。

1. **禁止修改原文件**：绝不能修改 `atoms/` 下现有的任何原子仿真器，也不能修改 `main.py` 和 `core/` 下的核心引擎（除了 `transforms.py` 允许追加）。需要复用但需微调时，请新建仿真器。
2. **原子绝对解耦**：新建的原子必须继承自 `core.base.AtomicSimulator`。原子的 `step(dt)` 内部**绝对不能**假设外部输入数据的来源、形状或单位，也不允许在原子内部做空间场采样或格式转换。输入端口直接读取 `self.inputs.get()`，只专注于本领域的纯粹计算。
3. **数据转换集中化 (Transform 协议)**：所有跨原子仿真器交互时的数据翻译，**必须且只能**写在 `core/transforms.py` 中。
   - 使用 `@register_transform` 装饰器追加在文件末尾。
   - 如果需要跨仿真器读取数据（如位置），必须通过传入二参函数的 `ctx: TransformContext` 进行读取（如 `ctx.sim_state('EC1', 'positions')`）。
4. **形式化的 YAML 拓扑**：必须创建一份全新的 `config/topology_xxx.yaml`，xxx为你给联合仿真环境简要起的英文称号。其中 `connections` 列表里的**每一条** S2S 连接，都必须显式声明 `transform` 字段，具体要求可以仿照已有的topology.yaml。
